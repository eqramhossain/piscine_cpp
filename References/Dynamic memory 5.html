Good — now we’re going to focus on dynamic string management in C++, but in the manual (C-style) way using new[] and delete[]. This is exactly where many bugs happen, so understanding it deeply is very valuable.


⸻


🔹 1) What is a “dynamic string”?


A dynamic string is simply:


A char* that points to memory allocated on the heap, ending with '\0'.


Example:


char *str = new char[6];


👉 This allocates space for 6 characters on the heap.


But right now, it is not yet a valid string, because:

• it contains garbage values

• no null terminator


⸻


🔹 2) Building a string manually


char *str = new char[6];


str[0] = 'h';

str[1] = 'e';

str[2] = 'l';

str[3] = 'l';

str[4] = 'o';

str[5] = '\0';


Now memory looks like:


[h][e][l][l][o][\0]


👉 Now it is a valid C-string.


⸻


🔹 3) The critical rule: always include space for '\0'


If your string has n characters:


👉 You must allocate n + 1


Example:


char *str = new char[5]; // ❌ wrong for "hello"


Because "hello" needs 6 bytes:

• 5 letters

• 1 null terminator


⸻


🔹 4) Copying strings dynamically (core skill)


This is one of the most important patterns.


Step-by-step logic:

1. Measure source length

2. Allocate memory

3. Copy characters

4. Add '\0'


Example:


#include <cstring>


char *src = (char *)"hello";


// 1. get length

size_t len = std::strlen(src);


// 2. allocate

char *copy = new char[len + 1];


// 3. copy

for (size_t i = 0; i < len; i++)

copy[i] = src[i];


// 4. null terminate

copy[len] = '\0';


👉 Now copy is an independent dynamic string.


⸻


🔹 5) Using strcpy (shortcut)


Instead of manual loop:


#include <cstring>


char *copy = new char[std::strlen(src) + 1];

std::strcpy(copy, src);


But ⚠️ you must guarantee:

• destination is large enough


⸻


🔹 6) Concatenating strings dynamically (harder)


This is where things get interesting.


Suppose:


char *a = (char *)"Hello ";

char *b = (char *)"World";


Think before coding:

• length of a = 6

• length of b = 5

• total = 11 + 1 ('\0')


👉 allocate 12


⸻


Implementation:


size_t lenA = std::strlen(a);

size_t lenB = std::strlen(b);


char *result = new char[lenA + lenB + 1];


// copy a

for (size_t i = 0; i < lenA; i++)

result[i] = a[i];


// copy b

for (size_t i = 0; i < lenB; i++)

result[lenA + i] = b[i];


// null terminate

result[lenA + lenB] = '\0';



⸻


🔹 7) Resizing a dynamic string


⚠️ You cannot resize directly. You must:

1. Allocate new memory

2. Copy old data

3. Delete old memory


Example idea:


char *str = new char[6]; // "hello"


// want to add "!"


Steps:

• allocate bigger array

• copy old content

• add new char

• delete old


⸻


Pattern:


size_t len = std::strlen(str);


char *newStr = new char[len + 2]; // +1 for '!' +1 for '\0'


// copy

for (size_t i = 0; i < len; i++)

newStr[i] = str[i];


// add new char

newStr[len] = '!';


// terminate

newStr[len + 1] = '\0';


// free old

delete[] str;


// update pointer

str = newStr;


👉 This pattern is fundamental.


⸻


🔹 8) Deleting dynamic strings


Always:


delete[] str;


Not:


delete str; // ❌ wrong


Because it was allocated with new[].


⸻


🔹 9) Common dangerous mistakes


❌ Forgetting '\0'


char *str = new char[5];


→ not a valid string


⸻


❌ Buffer overflow


char *str = new char[5];

std::strcpy(str, "hello"); // writes 6 bytes → overflow



⸻


❌ Memory leak


str = new char[10]; // lost previous pointer



⸻


❌ Modifying string literal


char *s = "hello";

s[0] = 'H'; // ❌ undefined behavior



⸻


🔹 10) Safe pattern (VERY important)


Whenever you handle dynamic strings, think:


👉 Allocate → Use → Free


Example:


char *str = new char[6];


// use it...


delete[] str;

str = nullptr;



⸻


🔹 11) Real-world mini example


Let’s simulate user input copy:


#include <iostream>

#include <cstring>


int main()

{

char buffer[100];

std::cin >> buffer;


char *str = new char[std::strlen(buffer) + 1];

std::strcpy(str, buffer);


std::cout << str << std::endl;


delete[] str;

}



⸻


🔹 12) Why this is hard (and important)


Dynamic strings are tricky because you must manually handle:

• memory size

• copying

• null terminator

• ownership

• deallocation


That’s why bugs like:

• leaks

• segmentation faults

• undefined behavior

are very common here.


⸻


🔹 13) How C++ solves this (preview)


All of this complexity:


char *str = new char[...];


is replaced by:


std::string str = "hello";

str += " world";


No manual allocation. No delete. No overflow risk.


⸻


🔹 14) Mental model you should keep


When working with dynamic strings:


“I am managing a block of memory that must:

• be big enough

• end with ‘\0’

• be copied manually

• be freed exactly once”
