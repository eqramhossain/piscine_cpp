👉 Parameterized constructors and copy constructors

These are fundamental in C++ and often misunderstood at the beginning.

⸻

🔹 1. Parameterized Constructor

A parameterized constructor lets you initialize an object with specific values at creation.

⸻

✅ Example

#include <iostream>
using namespace std;

class Student
{
public:
    string name;
    int age;

    // Parameterized constructor
    Student(string n, int a)
    {
        name = n;
        age = a;
    }
};

int main()
{
    Student s1("Alice", 20);
    Student s2("Bob", 25);

    cout << s1.name << " " << s1.age << endl;
    cout << s2.name << " " << s2.age << endl;
}


⸻

🧠 What happens here:

When you write:

Student s1("Alice", 20);

👉 C++ does:
	•	allocate memory for s1
	•	call constructor with "Alice" and 20
	•	initialize attributes

⸻

⚠️ Important Insight

If you define a parameterized constructor:
👉 the default constructor is NOT automatically created anymore

So this will break:

Student s3;  // ❌ ERROR (no default constructor)


⸻

🔹 2. Copy Constructor (Very Important)

Now we enter a core C++ concept.

A copy constructor is used when an object is created from another object.

⸻

👉 Syntax

ClassName(const ClassName& other);


⸻

✅ Simple Example

#include <iostream>
using namespace std;

class Student
{
public:
    string name;

    // Constructor
    Student(string n)
    {
        name = n;
    }

    // Copy constructor
    Student(const Student& other)
    {
        name = other.name;
        cout << "Copy constructor called!" << endl;
    }
};

int main()
{
    Student s1("Alice");
    Student s2 = s1;  // Copy constructor called

    cout << s2.name << endl;
}


⸻

🧠 When is the copy constructor called?

👉 In these cases:

Student s2 = s1;        // Copy initialization
Student s3(s1);        // Direct copy


⸻

🔥 3. VERY IMPORTANT (Shallow vs Deep Copy)

Now we connect this to your C knowledge.

⸻

❌ Problem Example (Shallow Copy)

class Buffer
{
public:
    int* data;

    Buffer()
    {
        data = new int(42);
    }

    ~Buffer()
    {
        delete data;
    }
};

Now:

Buffer b1;
Buffer b2 = b1;  // copy constructor (default)

👉 Both b1.data and b2.data point to same memory

⸻

💣 What happens?

When program ends:
	•	b1 destructor → delete data
	•	b2 destructor → ❌ delete same memory again → crash

⸻

✅ Solution: Deep Copy

Buffer(const Buffer& other)
{
    data = new int(*other.data);
}

👉 Now each object has its own memory

⸻

🔹 4. Why This Matters (Big Picture)

This leads to a fundamental C++ rule:

👉 Rule of Three

If your class uses:
	•	destructor
	•	dynamic memory (new)

Then you MUST implement:
	1.	Copy constructor
	2.	Copy assignment operator
	3.	Destructor

⸻

🔹 Mental Model (Very Important)

Think like this:

Concept	Meaning
Constructor	Create object
Copy constructor	Clone object
Destructor	Destroy object
