Let’s go step by step and really build your intuition about dynamic allocation in C++ using new and delete. I’ll keep it simple, but we’ll go deep enough so you actually understand what’s happening in memory.

🔹 1. Two Types of Memory (Core Idea)


Before even talking about new, you must understand this:


🧠 Stack (automatic memory)

  • Managed automatically

  • Fast

  • Variables disappear when leaving scope

int x = 5;

👉 x lives on the stack and is destroyed automatically.

🧠 Heap (dynamic memory)

  • Managed manually

  • Slower but flexible

  • You control lifetime


👉 This is where new comes in.

🔹 2. What is 

new

?


new allocates memory on the heap and returns a pointer to it.

int *ptr = new int;

What happens internally:

  1. Memory is reserved on the heap

  2. You get the address of that memory

  3. That address is stored in ptr


👉 Visualization:

Stack:        Heap:
ptr  -------> [ ? ]

🔹 3. Initializing with 

new

int *ptr = new int(42);

👉 Now:

ptr  -------> [ 42 ]

🔹 4. Why use dynamic allocation?


Because sometimes you don’t know the size at compile time:

int n;
std::cin >> n;

int *arr = new int[n];

👉 You can now create arrays with runtime size.

🔹 5. Dynamic Arrays

int *arr = new int[5];

Memory:

arr -------> [ ? ][ ? ][ ? ][ ? ][ ? ]

You use it like a normal array:

arr[0] = 10;
arr[1] = 20;

🔹 6. VERY IMPORTANT: 

delete


When you use new, you must free memory manually.

Single variable

int *ptr = new int(10);
delete ptr;

Array

int *arr = new int[5];
delete[] arr;

⚠️ Notice:

  • delete → single variable

  • delete[] → array

🔹 7. What happens if you don’t delete?


👉 You get a memory leak


Example:

void leak() {
    int *ptr = new int(5);
}

When the function ends:

  • ptr is destroyed ❌

  • heap memory still exists ❌


👉 You lost access → memory leak

🔹 8. Dangling Pointers (Danger)

int *ptr = new int(5);
delete ptr;

std::cout << *ptr; // ❌ undefined behavior

👉 ptr still points somewhere, but memory is invalid.

Safe practice:

delete ptr;
ptr = NULL; // or nullptr in modern C++

🔹 9. Dynamic Objects (Classes)

class Student {
public:
    std::string name;
};

Student *s = new Student;
s->name = "Alice";

delete s;

👉 Same logic, but for objects.

🔹 10. Multiple Levels (Pointer to Pointer)

int **ptr = new int*;
*ptr = new int(42);

Memory:

ptr -----> [ address ] -----> [ 42 ]

Freeing:

delete *ptr;
delete ptr;

🔹 11. Common Mistakes


❌ Forgetting delete


→ Memory leak


❌ Using wrong delete

int *arr = new int[5];
delete arr; // WRONG

❌ Double delete

delete ptr;
delete ptr; // ❌ crash risk

🔹 12. Think Like This (Key Mental Model)


When you write:

int *p = new int(10);

👉 You are saying:


“Give me a piece of memory that will NOT disappear automatically. I will manage it myself.”

🔹 13. Modern Note (Important for future)


In modern C++, we rarely use raw new/delete.


We use:

  • std::vector

  • std::unique_ptr

  • std::shared_ptr


But for learning (and 42 school), mastering new/delete is essential.
